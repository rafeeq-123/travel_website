Why do you think SSL/HTTPS is important?

Understanding why SSL is so important for a website is key to keeping your private information private. You would not want to put your credit card or social security number one a website that was not secure.


Explain, in a couple of paragraphs, what you think 2 major challenges around DNS configuration are for less-technical internet end-users?

I think that one of the biggest issues is security and understanding what an attacker can do and how they may do it. A lot of Paas platforms make it easy to do with a few commands, but what if a user does not want use one provider and they want to use multi-providers for example, a different host provider, a different domain name provider and so on (even if a user uses one provider they still should think about security). There are a lot of DNS security threats that I think that even non-tech savvy users should be aware of and how best to keep your website and computer more resistant to security threats, for example footprinting and DoS attacks.

Footprinting is an attack when an attacker gets “DNS zone data” that gives the attacker a lot of information with much of it being sensitive information. So if a user wants to take the steps to configure their own DNS it is wise to read up on ways to mitigate such attacks for instance, disable unneeded protocols and ports and “Lock down ports with the appropriate firewall configuration”, are I think doable with a little bit of google for many users. However damaging footprinting can be (some hackers just like to poke around), one can see how devastating the results can from DoS attack.

Very recently there was a massive DDoS attack and it opened the world’s eyes to how damaging such attacks can be and that even huge tech companies are not exempted. DoS attacks are when someone floods one or even sometimes more than one DNS server, pushing the CPUs to their maximum until they shut down. Nothing is really fool-proof, but understanding some basic ways to make the event of an attack less likely are making your website and or app with these types of attacks in mind and to stay up to speed with patches and security updates (of course there are more ways, but there again are things that most users can do). Though security is huge, there are other challenges too like understanding basic terminology can even hard at a very high level things.

Configuring DNS is intimidating things like what is happening your DNS Zone files can be overwhelming at first, but places like DigitalOcean have great guides for the terminology to better understand what's going on and why things like forwarding does and so on. Websites and tutorials can make basic DNS configuration really easy to understand, but I think a good understanding is pivotal and can be challenging for some people.
